# Antigravity Client (v1.0.16)

GUI-клиент и локальный прокси-сервер для интеграции и перенаправления запросов авторизации/API из **Antigravity 2.0** и **Antigravity IDE** на менеджер пула аккаунтов.

## 🏗 Архитектура запросов
```
IDE (VS Code / JetBrains) 
   --> Локальный прокси CLIента (порт 8047) 
   --> Antigravity-Manager (порт 8045 или 8055) 
   --> Google Backend
```

---

## 🚀 Что сделано в версии 1.0.16

1. **Исправление падения при первом запуске (Черный экран)**:
   * Добавлена задержка 1.5 секунды перед запуском IDE на ОС Windows. Это предотвращает возникновение ошибки тайм-аута загрузки (`ERR_TIMED_OUT` / черный экран), возникающей из-за того, что антивирус Windows Defender временно блокирует файлы `app.asar` и бинарники языкового сервера сразу после их патчинга.

2. **Оптимизация проксирования и стриминга (Streaming Proxy)**:
   * Реализовано селективное проксирование ответов. Потоковые эндпоинты автодополнения кода и чата (`loadCodeAssist`, `chat`) теперь проксируются напрямую через `StreamBody` (chunked encoding), исключая зависания и обрезание ответов.
   * Стандартные эндпоинты (такие как `fetchUserInfo`, `tokeninfo`) буферизуются полностью с явным указанием заголовка `Content-Length`, что предотвращает сбои аутентификации в IDE.

3. **Гарантированное восстановление состояния (Cleanup)**:
   * Вызов `restore_original_state_all()` теперь выполняется как при запуске приложения, так и при его корректном закрытии. Это гарантирует, что все файлы IDE возвращаются в исходное состояние, не ломая IDE при работе без запущенного клиента.

## 🛠 Запуск и разработка

Для запуска клиента в режиме разработки:
```bash
npm run tauri dev
```

Для сборки продакшн-версии:
```bash
npm run tauri build
```
